# Marketing assets — screenshots & video

Runbook for capturing demo media from [rinsehq.com](https://rinsehq.com) and publishing to the Rinse waitlist site.

## Folder layout

```
marketing/
  README.md                 ← this file
  demo-manifest.json        ← generated by seed:demo (IDs + routes)
  assets/stock-photos/      ← before/after JPGs for job photo seeding
  scripts/
    video-operator.md       ← 60–90s operator walkthrough beats
    video-product.md        ← 30–45s booking sync story
  raw/screenshots/          ← Playwright output (gitignored)
  raw/video/                ← screen recordings (gitignored)
  export/waitlist/          ← resized for detailingSignUP (gitignored)
  export/app-store/         ← framed App Store PNGs (gitignored)
```

## Environment

Add to `.env.local` (or export before running):

```bash
# PocketBase (production for rinsehq.com captures)
PB_URL=https://detailing-pb.fly.dev
PB_EMAIL=...
PB_PASSWORD=...

# Demo org — use atlas-detailing (existing) or summit-detail (fictional)
ORG_SLUG=atlas-detailing

# Next app URL for portal token + captures
APP_URL=https://rinsehq.com

# Portal token creation (from INTERNAL_API_SECRET)
INTERNAL_API_SECRET=...

# Optional: dedicated capture account (defaults to PB_EMAIL / PB_PASSWORD)
DEMO_CAPTURE_EMAIL=
DEMO_CAPTURE_PASSWORD=

# Optional: display name in app_settings during seed
DEMO_BUSINESS_NAME=Summit Mobile Detail
```

## Capture day checklist

1. **Re-seed** so relative dates are fresh:
   ```bash
   npm run seed:demo
   ```
2. Confirm `marketing/demo-manifest.json` has job/client IDs and `portalUrl`.
3. Sign in on a phone or simulator with the capture account.
4. Dismiss product tour if it appears (Playwright sets this automatically).
5. Run screenshots:
   ```bash
   npm run capture:screenshots
   ```
6. Record videos per `marketing/scripts/video-*.md` → save to `marketing/raw/video/`.
7. Export for waitlist (after raw assets exist):
   ```bash
   npm run export:waitlist-assets
   ```
8. Copy `marketing/export/waitlist/*` to `../detailingSignUP/assets/`.

## Stock photos

On first run, `npm run seed:demo` downloads open-source placeholders (Unsplash / Picsum) into `assets/stock-photos/`. Credits are listed in `SOURCES.md`.

**Swap in your own photos:** replace any JPG in that folder (keep filenames or update `scripts/lib/demo-images.mjs`), then:

```bash
FORCE_DEMO_PHOTOS=1 npm run seed:demo
```

`FORCE_DEMO_PHOTOS=1` replaces job gallery images; without it, existing job photos are left alone.
